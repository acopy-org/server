// how it works

the client runs as a background service. it monitors your clipboard and syncs changes to all your other devices over a persistent websocket connection. clipboard data is encrypted in transit and stored only for history.

// headless & SSH

on headless Linux (e.g. over SSH), there's no display server to access the clipboard. acopy detects this automatically and uses OSC 52 — a terminal escape sequence that tells your local terminal emulator to set its clipboard. the text travels through your SSH connection back to the terminal on your machine, no X11 forwarding needed.

for images, OSC 52 only supports text, so acopy copies a link to the image on the server (e.g. https://acopy.org/c/<id>) into your clipboard instead.

works with terminals that support OSC 52: iTerm2, kitty, alacritty, foot, Windows Terminal, and others.
